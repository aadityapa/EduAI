# Customer Success Workflow — Beta

---

## Team Roles

| Role | Responsibility |
|------|----------------|
| CSM | Primary school relationship, weekly check-ins |
| Implementation Specialist | Roster import, branding, training schedule |
| Support L1 | Ticket triage, password resets |
| Product Liaison | Escalations, feedback synthesis |

---

## Lifecycle Stages

```
Prospect → Signed → Onboarding → Training → Go-Live → Active → Renewal
```

---

## Onboarding Timeline (21 days)

| Week | CSM Actions |
|------|-------------|
| 0 | Kickoff call; share onboarding guide |
| 1 | Daily standup with school IT; roster validation |
| 2 | Training sessions; monitor login rates |
| 3 | Go-live support; first weekly report to admin |

---

## Health Score (Beta)

| Metric | Green | Yellow | Red |
|--------|-------|--------|-----|
| Weekly active students | >70% | 40–70% | <40% |
| Teacher login (weekly) | >80% | 50–80% | <50% |
| Open P1/P2 tickets | 0 | 1–2 | 3+ |
| NPS | >40 | 20–40 | <20 |

Red health → executive escalation + recovery plan within 72h.

---

## Deliverables

- **Week 1 report:** Login stats, blockers
- **Month 1 report:** Usage analytics, NPS, content gaps, renewal recommendation

---

## Renewal (Pilot → Paid)

60 days before pilot end:
1. Review health score
2. Present ROI deck (time saved, engagement)
3. Propose subscription tier
4. Legal: DPA + MSA if not signed
